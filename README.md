# 1 Введение

<a name="appointment"/>

## 1.1 Назначение
В этом документе описаны функциональные и нефункциональные требования к приложению «Мониторинг продуктивности сотрудников» для ОС Windows 10. Этот документ предназначен для команды, которая будет реализовывать и проверять корректность работы приложения.

<a name="business_requirements"/>

## 1.2 Бизнес-требования

<a name="initial_data"/>

### 1.2.1 Исходные данные
В процессе осуществления проекта необходимо иметь средства контроля продуктивности сотрудников, позволяющие оценить эффективность работы команды, так и отдельных лиц. Необходимость внедрения подобных инструментов для контроля эффективности работы персонала, обусловлена человеческим фактором.

<a name="business_opportunities"/>

### 1.2.2 Возможности бизнеса
Это обеспечит надежность и уверенность в высокой производительности сотрудников на всех стадиях разработки проекта и позволит избежать непредвиденных ситуаций, ставящих под сомнение своевременную сдачу проекта. Приложение позволит отслеживать проблемы, связанные с организацией и контролем деятельности сотрудников, получать более высокие показатели слаженности и эффективности рабочего процесса.

<a name="project_boundary"/>

### 1.2.3 Границы проекта
Приложение «Мониторинг продуктивности сотрудников» позволит руководителям проектов получать еженедельно информацию о успехах и неудачах сотруников, задействованнных в реализации проектов. Сотрудники получат возможность немедленного реагирования на проблемы, возникающие в ходе работы, и получения четких, конкретно поставленных задач.

<a name="weekly_planning"/>

#### 3.1.1.2 Еженедельное планирование
**Описание.** Зарегистрированный пользователь имеет возможность отмечать успехи, неудачи и планы в своей еженедельной трудовой деятельности.
 
| Функция | Требования | 
|:---|:---|
| Добавление планов, успехов, неудач | Приложение должно предоставить зарегистрированному пользователю поле для ввода описания плана, неудачи, успеха. Пользователь должен либо ввести информацию и подвердить действие, либо отменить его |
| Удаление планов, успехов, неудач | Зарегистрированный пользователь имеет возможность удаления планов, успехов, и неудач |
| Коррекция описания планов, успехов, неудач | Приложение должно предоставить зарегистрированному пользователю возможность скорректировать описание |

<a name="quarterly_planning"/>

#### 3.1.1.3 Квартальное планирование
**Описание.** Зарегистрированный пользователь имеет возможность отмечать квартальные цели.

| Функция | Требования | 
|:---|:---|
| Добавление квартальных целей | Приложение должно предоставить зарегистрированному пользователю поле для ввода квартальной цели. Пользователь должен либо ввести информацию и подвердить действие, либо отменить его |
| Удаление квартальных целей | Зарегистрированный пользователь имеет возможность удаления квартальных целей |
| Коррекция описания квартальных целей | Приложение должно предоставить зарегистрированному пользователю возможность скорректировать информацию |
